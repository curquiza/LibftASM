<p>Almost all programming languages have the ability to change the order in which statements are evaluated, and assembly is no exception. The instruction pointer (EIP) register contains the address of the next instruction to be executed. To change the flow of control, the programmer must be able to modify the value of EIP. This is where control flow functions come in.
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="nf">mov</span> <span class="no">eip</span><span class="p">,</span> <span class="no">label</span>   <span class="c">; wrong</span>
<span class="no">jmp</span> <span class="no">label</span>        <span class="c">; right</span>
</pre></div>
<p><br />
</p>

<h2><span class="mw-headline" id="Comparison_Instructions">Comparison Instructions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=1" title="Edit section: Comparison Instructions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<table width="100%" class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">

<tbody><tr>
<td style="padding-left: 10px;"><big> <b>test</b> arg0, arg1</big>
</td>
<td width="75px"><small><b><a href="/wiki/X86_Assembly/GAS_Syntax" title="X86 Assembly/GAS Syntax">GAS Syntax</a></b></small>
</td></tr>
<tr>
<td style="padding-left: 10px;"><big> <b>test</b> arg1, arg0</big>
</td>
<td><small><b><a href="/wiki/X86_Assembly/MASM_Syntax" title="X86 Assembly/MASM Syntax">Intel syntax</a></b></small>
</td></tr></tbody></table>
<p><br />
Performs a bit-wise logical <code>and</code> on <code>arg0</code> and <code>arg1</code> the result of which we will refer to as <code>commonBits</code> and sets the <code>ZF</code>(zero), <code>SF</code>(sign) and <code>PF</code> (parity) flags based on <code>commonBits</code>. <code>CommonBits</code> is then discarded.
</p><p><br />
<b> Operands </b>
</p><p><i>arg0</i>
</p>
<ul><li>Register</li>
<li>Immediate</li></ul>
<p><i>arg1</i>
</p>
<ul><li><code>AL/AX/EAX</code> (only if <code>arg0</code> is an immediate value)</li>
<li>Register</li>
<li>Memory</li></ul>
<p><b> Modified flags </b>
</p>
<ul><li><code>SF</code>&#160;≔&#160;MostSignificantBit(<code>commonBits</code>)</li>
<li><code>ZF</code>&#160;≔ (<code>commonBits</code> = 0), so a set <code>ZF</code> means, <code>arg0</code> and <code>arg1</code> do not have any set bits in common</li>
<li><code>PF</code>&#160;≔&#160;BitWiseXorNor(<code>commonBits</code>[Max-1:0]), so <code>PF</code> is set if and only if <code>commonBits</code>[Max-1:0] has an even number of 1 bits</li>
<li><code>CF</code>&#160;≔&#160;0</li>
<li><code>OF</code>&#160;≔&#160;0</li>
<li><code>AF</code> is undefined</li></ul>
<p><b> Application </b>
</p>
<ul><li>passing the same register twice: <code>test rax, rax</code>
<ul><li><code>SF</code> becomes the <i>sign</i> of <code>rax</code>, which is a simple test for non-negativity</li>
<li><code>ZF</code> is set if <code>rax</code> is <i>zero</i></li>
<li><code>PF</code> is set if <code>rax</code> has an even number of set bits</li></ul></li></ul>
<p><br />
</p>
<table width="100%" class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">

<tbody><tr>
<td style="padding-left: 10px;"><big> <b>cmp</b> subtrahend, minuend</big>
</td>
<td width="75px"><small><b><a href="/wiki/X86_Assembly/GAS_Syntax" title="X86 Assembly/GAS Syntax">GAS Syntax</a></b></small>
</td></tr>
<tr>
<td style="padding-left: 10px;"><big> <b>cmp</b> minuend, subtrahend</big>
</td>
<td><small><b><a href="/wiki/X86_Assembly/MASM_Syntax" title="X86 Assembly/MASM Syntax">Intel syntax</a></b></small>
</td></tr></tbody></table>
<p><br />
Performs a comparison operation between <code>minuend</code> and <code>subtrahend</code>. The comparison is performed by a (signed) subtraction of <code>subtrahend</code> from <code>minuend</code>, the results of which can be called <code>difference</code>. <code>Difference</code> is then discarded. If <code>subtrahend</code> is an immediate value it will be sign extended to the length of <code>minuend</code>. The <code>EFLAGS</code> register is set in the same manner as a <code>sub</code> instruction.
</p><p>Note that the GAS/AT&amp;T syntax can be rather confusing, as for example <code>cmp $0, %rax</code> followed by <code>jl branch</code> will branch if <code>%rax &lt; 0</code> (and not the opposite as might be expected from the order of the operands).
</p><p><br />
<b> Operands </b>
</p><p><i>minuend</i>
</p>
<ul><li><code>AL/AX/EAX</code> (only if <code>subtrahend</code> is immediate)</li>
<li>Register</li>
<li>Memory</li></ul>
<p><i>subtrahend</i>
</p>
<ul><li>Register</li>
<li>Immediate</li>
<li>Memory</li></ul>
<p><br />
<b> Modified flags </b>
</p>
<ul><li><code>SF</code>&#160;≔&#160;MostSignificantBit(<code>difference</code>), so a unset <code>SF</code> means the <code>difference</code> is non-negative (<code>minuend</code>&#160;≥&#160;<code>subtrahend</code> [NB: signed comparison])</li>
<li><code>ZF</code>&#160;≔&#160;(<code>difference</code> =&#160;0)</li>
<li><code>PF</code>&#160;≔&#160;BitWiseXorNor(<code>difference</code>[Max-1:0])</li>
<li><code>CF</code>, <code>OF</code> and <code>AF</code></li></ul>
<h2><span class="mw-headline" id="Jump_Instructions">Jump Instructions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=2" title="Edit section: Jump Instructions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The jump instructions allow the programmer to (indirectly) set the value of the EIP register. The location passed as the argument is usually a label. The first instruction executed after the jump is the instruction immediately following the label. All of the jump instructions, with the exception of <code>jmp</code>, are <b>conditional jumps</b>, meaning that program flow is diverted only if a condition is true. These instructions are often used after a comparison instruction (see above), but since many other instructions set flags, this order is not required.
</p><p>See <a href="/wiki/X86_Assembly/X86_Architecture#EFLAGS_Register" title="X86 Assembly/X86 Architecture">chapter “X86 architecture”, §&#160;“EFLAGS register”</a> for more information about the flags and their meaning.
</p>
<h3><span class="mw-headline" id="Unconditional_Jumps">Unconditional Jumps</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=3" title="Edit section: Unconditional Jumps">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jmp</b> loc </big>
</p>
</div>
<p>Loads <code>EIP</code> with the specified address (i.e. the next instruction executed will be the one specified by <code>jmp</code>).
</p>
<h3><span class="mw-headline" id="Jump_if_Equal">Jump if Equal</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=4" title="Edit section: Jump if Equal">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>je</b> loc </big>
</p>
</div>
<p><i>ZF = 1</i>
</p><p>Loads <code>EIP</code> with the specified address, if operands of previous <code>cmp</code> instruction are equal.
For example:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="nf">mov</span> <span class="no">$5</span><span class="p">,</span> <span class="no">ecx</span>
<span class="nf">mov</span> <span class="no">$5</span><span class="p">,</span> <span class="no">edx</span>
<span class="nf">cmp</span> <span class="no">ecx</span><span class="p">,</span> <span class="no">edx</span>
<span class="nf">je</span> <span class="no">equal</span>
<span class="c">; if it did not jump to the label equal,</span>
<span class="c">; then this means ecx and edx are not equal.</span>
<span class="nl">equal:</span>
<span class="c">; if it jumped here, then this means ecx and edx are equal</span>
</pre></div>
<h3><span class="mw-headline" id="Jump_if_Not_Equal">Jump if Not Equal</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=5" title="Edit section: Jump if Not Equal">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jne</b> loc </big>
</p>
</div>
<p><i>ZF = 0</i>
</p><p>Loads <code>EIP</code> with the specified address, if operands of previous <code>cmp</code> instruction are not equal.
</p>
<h3><span class="mw-headline" id="Jump_if_Greater">Jump if Greater</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=6" title="Edit section: Jump if Greater">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jg</b> loc </big>
</p>
</div>
<p><i>SF = OF and ZF = 0</i>
</p><p>Loads <code>EIP</code> with the specified address, if the <code>minuend</code> of the previous <code>cmp</code> instruction is greater than the second (performs signed comparison).
</p>
<h3><span class="mw-headline" id="Jump_if_Greater_or_Equal">Jump if Greater or Equal</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=7" title="Edit section: Jump if Greater or Equal">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jge</b> loc </big>
</p>
</div>
<p><i>SF = OF or ZF = 1</i>
</p><p>Loads <code>EIP</code> with the specified address, if the <code>minuend</code> of the of previous <code>cmp</code> instruction is greater than or equal to the <code>subtrahend</code> (performs signed comparison).
</p>
<h3><span id="Jump_if_Above_.28unsigned_comparison.29"></span><span class="mw-headline" id="Jump_if_Above_(unsigned_comparison)">Jump if Above (unsigned comparison)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=8" title="Edit section: Jump if Above (unsigned comparison)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>ja</b> loc </big>
</p>
</div>
<p><i>CF = 0 and ZF = 0</i>
</p><p>Loads <code>EIP</code> with the specified address, if the <code>minuend</code> of the previous <code>cmp</code> instruction is greater than the <code>subtrahend</code>.
<code>ja</code> is the same as <code>jg</code>, except that it performs an unsigned comparison.
</p><p>That means, the following piece of code <i>always</i> jumps (unless <code>rbx</code> is <code>-1</code>, too), because negative one is represented as all bits set in the two's complement.
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="nf">mov</span> <span class="no">rax</span><span class="p">,</span> <span class="p">-</span><span class="mi">1</span> <span class="err">//</span> <span class="no">rax</span> <span class="p">:</span><span class="err">=</span> <span class="p">-</span><span class="mi">1</span>
<span class="nf">cmp</span> <span class="no">rax</span><span class="p">,</span> <span class="no">rbx</span>
<span class="nf">ja</span> <span class="no">loc</span>
</pre></div>
<p>Interpreting all bits set (without treating any bit as the sign) has the value 2ⁿ-1 (where n is the length of the register).
That is the largest unsigned value a register can hold.
</p>
<h3><span id="Jump_if_Above_or_Equal_.28unsigned_comparison.29"></span><span class="mw-headline" id="Jump_if_Above_or_Equal_(unsigned_comparison)">Jump if Above or Equal (unsigned comparison)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=9" title="Edit section: Jump if Above or Equal (unsigned comparison)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jae</b> loc </big>
</p>
</div>
<p><i>CF = 0 or ZF = 1</i>
</p><p>Loads <code>EIP</code> with the specified address, if the <code>minuend</code> of previous <code>cmp</code> instruction is greater than or equal to the <code>subtrahend</code>.
<code>jae</code> is the same as <code>jge</code>, except that it performs an unsigned comparison.
</p>
<h3><span class="mw-headline" id="Jump_if_Lesser">Jump if Lesser</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=10" title="Edit section: Jump if Lesser">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jl</b> loc </big>
</p>
</div>
<p>The criterion required for a <code>jl</code> is that <code>SF</code>&#160;≠&#160;<code>OF</code>.
It loads <code>EIP</code> with the specified address, if the criterion is met.
So either <code>SF</code> or <code>OF</code> can be set, but not both in order to satisfy this criterion.
If we take the <code>sub</code> (which is <i>basically</i> what a <code>cmp</code> does) instruction as an example, we have:
</p>
<dl><dd><code>minuend</code>&#160;-&#160;<code>subtrahend</code></dd></dl>
<p>With respect to <code>sub</code> and <code>cmp</code> there are several cases that fulfill this criterion:
</p>
<ol><li><code>minuend</code> &lt; <code>subtrahend</code> and the operation does not have overflow</li>
<li><code>minuend</code> &gt; <code>subtrahend</code> and the operation has an overflow</li></ol>
<p><br />
In the first case <code>SF</code> will be set but not <code>OF</code> and in second case <code>OF</code> will be set but not <code>SF</code> since the overflow will reset the most significant bit to zero and thus preventing <code>SF</code> being set.
The <code>SF</code>&#160;≠&#160;<code>OF</code> criterion avoids the cases where:
</p>
<ol><li><code>minuend</code>&#160;&gt;&#160;<code>subtrahend</code> and the operation does not have overflow</li>
<li><code>minuend</code>&#160;&lt;&#160;<code>subtrahend</code> and the operation has an overflow</li>
<li><code>minuend</code>&#160;=&#160;<code>subtrahend</code></li></ol>
<p>In the first case neither <code>SF</code> nor <code>OF</code> are set, in the second case <code>OF</code> will be set and <code>SF</code> will be set since the overflow will reset the most significant bit to one and in the last case neither <code>SF</code> nor <code>OF</code> will be set.
</p>
<h3><span class="mw-headline" id="Jump_if_Less_or_Equal">Jump if Less or Equal</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=11" title="Edit section: Jump if Less or Equal">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jle</b> loc </big>
</p>
</div>
<p><code>SF</code>&#160;≠&#160;<code>OF</code> or <code>ZF = 1</code>. 
</p><p>Loads <code>EIP</code> with the specified address, if the <code>minuend</code> of previous <code>cmp</code> instruction is lesser than or equal to the <code>subtrahend</code>.
See the <a href="#Jump_if_Lesser"><code>jl</code> section</a> for a more detailed description of the criteria.
</p>
<h3><span id="Jump_if_Below_.28unsigned_comparison.29"></span><span class="mw-headline" id="Jump_if_Below_(unsigned_comparison)">Jump if Below (unsigned comparison)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=12" title="Edit section: Jump if Below (unsigned comparison)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jb</b> loc </big>
</p>
</div>
<p><i>CF = 1</i>
</p><p>Loads EIP with the specified address, if first operand of previous CMP instruction is lesser than the second. <code>jb</code> is the same as <code>jl</code>, except that it performs an unsigned comparison.
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="nf">mov</span> <span class="no">rax</span><span class="p">,</span> <span class="mi">0</span>   <span class="c">; rax&#160;≔&#160;0</span>
<span class="no">cmp</span> <span class="no">rax</span><span class="p">,</span> <span class="no">rbx</span> <span class="c">; rax&#160;≟&#160;rbx</span>
<span class="no">jb</span> <span class="no">loc</span>       <span class="c">; always jumps, unless rbx is also 0</span>
</pre></div>
<h3><span id="Jump_if_Below_or_Equal_.28unsigned_comparison.29"></span><span class="mw-headline" id="Jump_if_Below_or_Equal_(unsigned_comparison)">Jump if Below or Equal (unsigned comparison)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=13" title="Edit section: Jump if Below or Equal (unsigned comparison)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>jbe</b> loc </big>
</p>
</div>
<p><i>CF = 1 or ZF = 1</i>
</p><p>Loads EIP with the specified address, if <code>minuend</code> of previous <code>cmp</code> instruction is lesser than or equal to the <code>subtrahend</code>.
<code>jbe</code> is the same as <code>jle</code>, except that it performs an unsigned comparison.
</p>
<h3><span class="mw-headline" id="Jump_if_Overflow">Jump if Overflow</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=14" title="Edit section: Jump if Overflow">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<h2><span class="mw-headline" id="Function_Calls">Function Calls</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=21" title="Edit section: Function Calls">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>call</b> proc </big>
</p>
</div>
<p>Pushes the address of the instruction that follows the <code>call</code> call, i.e. usually the next line in your source code, onto the top of the stack, and then jumps to the specified location.
This is used mostly for subroutines.
</p>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>ret</b> [val] </big>
</p>
</div>
<p>Loads the next value on the stack into <code>EIP</code>, and then pops the specified number of bytes off the stack.
If <code>val</code> is not supplied, the instruction will not pop any values off the stack after returning.
</p>
<h2><span class="mw-headline" id="Loop_Instructions">Loop Instructions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=22" title="Edit section: Loop Instructions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>loop</b> arg </big>
</p>
</div>
<p>The <code>loop</code> instruction decrements <code>ECX</code> and jumps to the address specified by <code>arg</code> unless decrementing <code>ECX</code> caused its value to become zero.
For example:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span>	<span class="nf">mov</span> <span class="no">ecx</span><span class="p">,</span> <span class="mi">5</span> <span class="c">; ecx ≔ 5</span>
<span class="no">head</span><span class="p">:</span>
	<span class="c">; the code here would be executed 5 times</span>
	<span class="nf">loop</span> <span class="no">head</span>
</pre></div>
<p><code>loop</code> does not set any flags.
</p>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>loopcc</b> arg </big>
</p>
</div>
<p>These loop instructions decrement <code>ECX</code> and jump to the address specified by <code>arg</code> if their condition is satisfied (that is, a specific flag is set), unless decrementing <code>ECX</code> caused its value to become zero.
</p>
<ul><li><code>loope</code>  loop if equal</li>
<li><code>loopne</code> loop if not equal</li>
<li><code>loopnz</code> loop if not zero</li>
<li><code>loopz</code>  loop if zero</li></ul>
<p>That way, only testing for a non-zero <code>ECX</code> can be combined with testing <code>ZF</code>.
Other flags can not be tested for, say there is no <code>loopnc</code> “loop while <code>ECX</code>&#160;≠&#160;0 and <code>CF</code> unset”.
</p>
<h2><span class="mw-headline" id="Enter_and_Leave">Enter and Leave</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=23" title="Edit section: Enter and Leave">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>enter</b> arg </big>
</p>
</div>
<p><code>enter</code> creates a stack frame with the specified amount of space allocated on the stack.
</p>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>leave</b> </big>
</p>
</div>
<p><code>leave</code> destroys the current stack frame, and restores the previous frame.
Using Intel syntax this is equivalent to:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="nf">mov</span> <span class="no">esp</span><span class="p">,</span> <span class="no">ebp</span> <span class="c">; esp ≔ ebp</span>
<span class="no">pop</span> <span class="no">ebp</span>
</pre></div>
<p>This will set <code>EBP</code> and <code>ESP</code> to their respective value before the function prologue began therefore reversing any modification to the stack that took place during the prologue.
</p>
<h2><span class="mw-headline" id="Other_Control_Instructions">Other Control Instructions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=X86_Assembly/Control_Flow&amp;action=edit&amp;section=24" title="Edit section: Other Control Instructions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>hlt</b> </big>
</p>
</div>
<p>Halts the processor.
Execution will be resumed after processing next hardware interrupt, unless <code>IF</code> is cleared.
</p>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>nop</b> </big>
</p>
</div>
<p>No operation.
This instruction doesn't do anything, but wastes (an) instruction cycle(s) in the processor.
</p><p>This instruction is often <i>represented</i> as an <code>xchg</code> operation with the operands <code>EAX</code> and <code>EAX</code> (an operation without side-effects), because there is no designated opcode for doing nothing.
This just as a passing remark, so that you do not get confused with <i>disassembled</i> code.
</p>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>lock</b> </big>
</p>
</div>
<p>Asserts #LOCK prefix on next instruction.
</p>
<div class="PrettyTextBox" style="padding-left: 10px; padding-top: 2px; padding-bottom: 2px; float: center; clear: both;">
<p><big> <b>wait</b> </big>
</p>
</div>
<p>Waits for the FPU to finish its last calculation.
</p>
